# Response Relay (Demo)

Safe response placeholder: relay/MOSFET to LED/lamp/dummy load only. No launcher or harmful hardware.

## Description and demo goal

Receives detect/decision signal (e.g. MQTT); drives a safe output (relay or MOSFET to LED, lamp, or dummy load). Arm/disarm and log activations. Used to validate the response path without any launcher.

## Comparison (1a / 1b / 1c)

| Option | Description | Cost | Reliability | Best for |
|--------|-------------|------|-------------|----------|
| **1a** | Basic relay module, minimal circuit; arm/disarm via GPIO. | Lowest | Adequate | Lab only |
| **1b** | Relay or MOSFET with status LED; optional MQTT logging. | Medium | Good | Default demo |
| **1c** | Relay + diagnostics (current sense, log to SD/cloud); mTLS or token auth. | Higher | Best | Logging and diagnostics |

## BOM (outline)

- Relay or MOSFET module
- Dummy load (e.g. LED, lamp)
- Switch (arm/disarm)
- Status LED
- ESP32 (or equivalent) for control and MQTT

No launcher or harmful hardware. See [safety boundary](../safety-boundary.mdx).

## How-to

Wiring, flash, config (MQTT topic, arm/disarm logic), test. Link to [common/actuator](../../common/actuator.mdx), [common/networking](../../common/networking.mdx).
