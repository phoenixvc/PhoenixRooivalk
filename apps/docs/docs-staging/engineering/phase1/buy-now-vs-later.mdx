# Phase-1 Buy Now vs Later

Summary of what to **buy now** (Phase 1a / cheapest demo) vs **leave for later**
(1b/1c and Phase 2) for each Phase-1 product. Rationale is kept short; details
and BOMs are in each [product doc](./products/index.mdx) and
[Platform BOM v1](./platform-bom-v1.mdx).

**Design principle:** Minimal now, refactor later. Shared baseline below;
product-specific rows follow. For per-product build steps, wiring, and “why buy
now vs later”, see each product doc and [Platform BOM v1](./platform-bom-v1.mdx)
(including pre-checkout and remove-before-order).

## Shared baseline

| Buy now (shared)                                                         | Leave for later (shared)                              | Why                                                                                               |
| ------------------------------------------------------------------------ | ----------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| ESP32 dev board, buck (MP1584EN), jumper wires, buzzer, LED, basic mount | IP enclosure, cable glands, LTE/PoE, formal harnesses | Phase 1a needs a working demo; ruggedization and long-range add cost without improving core demo. |

## Rationale (short)

- **Nano / Standard:** Prove detect → alert loop first; add Pi+Coral when budget
  allows for standalone inference.
- **Mesh:** Prove multi-node and central aggregation with minimal nodes; add PoE
  and dedicated central later.
- **Turret:** Servo pan/tilt + alarm only; no solenoid/stepper in Phase 1. Add
  turret camera only if needed for closed-loop or UI polish.
- **Handheld:** Portable detection + alert; phone or ESP32-CAM for 1a.
- **Response Relay:** Safe output only (relay/MOSFET to LED/lamp); no launcher
  or harmful hardware.
