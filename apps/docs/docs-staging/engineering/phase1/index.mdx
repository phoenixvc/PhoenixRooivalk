# Phase-1 Demo Builds

Shared build blocks (power, networking, actuator, compute, alarms) live in [Common modules](../common/index.mdx). Phase-1 stack and BOMs are described in this section and in each product doc.

Phase-1 is a **demoable capability set**, not production SKUs. It covers detection, tracking, alarm, and inert actuation only. Build docs and BOMs here support ultra-cheap (1a), balanced (1b), and better (1c) options per product.

## Purpose

- Demoable end-to-end flows: detection → tracking → alarm and safe actuation (relay/LED or pan/tilt).
- Shared platform: Pi4+Coral, MQTT, ESP32 actuator; common modules for power, alarms, networking, actuator, compute.
- Clear safety boundary and avoid list so builds stay within scope.

## Comparison: ultra-cheap / balanced / better

Across products, options are labeled **1a** (ultra-cheap), **1b** (balanced), **1c** (better). Each product doc has a comparison table and weighted matrix (cost, performance, reliability, effort) with a short choice summary. Scrapyard variants are optional where noted.

## Evolution: Phase 0 → 1 → 2

- **Phase 0:** Minimal or blind actuator; no turret camera.
- **Phase 1:** Optional turret camera (ESP32-CAM or USB to Pi); full BOMs and build docs in this section.
- **Phase 2:** Dual camera, stiffer mechanics; out of scope for this doc set.

Canonical stack: Pi4+Coral for detection/inference, MQTT as event spine, ESP32 as actuator controller, PCA9685 for servos.

## Safety boundary

Phase 1 is **detection, tracking, alarm, and inert actuation only**. No weaponization.

- **Response Relay:** Relay/MOSFET to LED/lamp/dummy load only; arm/disarm; log activations. No launcher or harmful hardware.
- **Turret:** Pan/tilt + alarm only. No firing hardware.
- **All products:** No building or operating systems intended to shoot down drones; no detailed firing hardware in docs.

See [Safety boundary](./safety-boundary.mdx) for the single reference (or in-doc summary above).

## Shared parts

Common parts across many Phase-1 builds: ESP32, MP1584 (buck), jumper wires, buzzer, LED. See [Shared parts](../common/shared-parts.mdx) for a consolidated list.

## Wiring safety

- **Common GND:** Servo PSU GND must tie to ESP32 GND.
- **Set voltage before connecting:** Adjust buck output with no load before connecting ESP32 or servos.
- **Fuse / power switch** on supply side where appropriate.
- **Bulk capacitor (470–1000 µF)** on the servo rail.

Details: [common/power](../common/power.mdx) and [Wiring safety](../common/wiring-safety.mdx).

## Phase 1 avoid list

Do **not** use the following for Phase 1 builds as specified:

| Avoid | Use instead |
|-------|-------------|
| L298N for servos | PCA9685 |
| OV7670 on ESP32 | OV2640 (e.g. ESP32-CAM) |
| 360° continuous-rotation servos for pan/tilt | 180° positional servos |
| L293D for servos | PCA9685 |
| Brushless motors, ESCs, flight controller, XT60 PDB for turret | Servo-based pan/tilt only |
| Duplicate IMU families; TCA9548A until needed | Per product BOM |

Full list: [Avoid list](./avoid-list.mdx).

## Platform BOM v1

**Architecture lock:** ESP32 + RP2040 modular robotics stack; PCA9685, TB6612/BTS7960 for motor/servo control.

- **Buy now:** Core compute, one actuator stack, minimal sensing.
- **Buy soon:** Spares, second actuator stack, better camera.
- **Buy later:** Mesh radio, extra storage, enclosures.

Quantities cheat sheet and tier choices are in each product doc. Optional: [Platform BOM v1](./platform-bom-v1.mdx).

## Buy now vs later

Per product, a short “buy now (1a)” vs “leave for later (1b/1c)” with brief rationale. See each [product doc](./products/index.mdx) for that product’s table or subsection.

## Budget allocation (optional)

If you have ~R2000 to spend, a typical priority order: power + compute spares → actuation stack → motor drivers → sensing → camera. Adjust per product and local pricing.

## Architectural decisions (Phase-1)

Six decisions are reflected in the docs; they will be formalized as ADRs later:

1. Event bus (e.g. MQTT backbone)
2. Compute baseline (Pi4+Coral default)
3. Alarm strategy (local + remote)
4. Actuator separation (ESP32, PCA9685)
5. Doc structure (this Phase-1 area + common modules)
6. Canonical hardware (platform BOM v1)
