# Slide Deck Components Guide

This guide covers all the slide deck functionality available in the Phoenix Rooivalk documentation system. These components enable rich, interactive presentations that can be viewed inline, printed, exported to PowerPoint, or presented in full-screen mode.

---

## Quick Start

```tsx
import { SlideDeck, SlideSection } from "@site/src/components/Downloads";

<SlideDeck title="My Presentation" duration={5} audience="Investors">
  <SlideSection
    number={1}
    title="Introduction"
    duration={60}
    icon="üéØ"
    keyPoints={["First point", "Second point"]}
    script="Speaker notes go here..."
  />
</SlideDeck>
```

## Rich Text Formatting

Use markdown-style formatting in key points for visual emphasis:

```tsx
keyPoints={[
  "**Bold text** for emphasis",
  "*Italic text* for subtle emphasis",
  "Combined **bold** and *italic* formatting",
  "Regular text without formatting"
]}
```

### Rendering

- `**text**` renders as **bold**
- `*text*` renders as *italic*
- Formatting works in key points, column content, and nested bullets

---

## Nested Bullets (Sub-Points)

For complex information, use `KeyPoint` objects with sub-points:

```tsx
keyPoints={[
  {
    text: "**Main point with details**",
    subPoints: [
      "First supporting detail",
      "Second supporting detail",
      "*Italicized* sub-point"
    ]
  },
  "Simple string point (no sub-bullets)"
]}
```

Sub-points render as indented secondary bullets beneath the main point.

## Slide Layouts

### Default Layout

Standard bullet-point slide with key points.

```tsx
<SlideSection
  number={1}
  title="Standard Slide"
  keyPoints={["Point 1", "Point 2", "Point 3"]}
/>
```

### Title-Only Layout

Section dividers or emphasis slides with minimal content.

```tsx
<SlideSection
  number={1}
  title="Section Title"
  layout="title-only"
  keyPoints={["Optional subtitle or tagline"]}
/>
```

### Two-Column Layout

Side-by-side comparisons (great for vs. competitors, before/after, etc.).

```tsx
<SlideSection
  number={1}
  title="Comparison"
  layout="two-column"
  leftColumnTitle="Our Solution"
  leftColumn={[
    "**120-195ms** response",
    "‚úÖ Full offline capability",
    "‚úÖ Blockchain evidence"
  ]}
  rightColumnTitle="Competitors"
  rightColumn={[
    "**2-5 seconds** response",
    "‚ùå Cloud dependent",
    "‚ùå No evidence trail"
  ]}
  keyPoints={[]} // Empty for pure two-column
/>
```

### Quote Layout

For testimonials or impactful quotes.

```tsx
<SlideSection
  number={1}
  title="What Customers Say"
  layout="quote"
  keyPoints={["This solution changed everything for our airport security."]}
  quoteAuthor="Airport Security Director"
/>
```

### Code Layout

For technical presentations with code examples.

```tsx
<SlideSection
  number={1}
  title="API Example"
  layout="code"
  codeBlock={`const response = await fetch('/api/detect');
const threats = await response.json();`}
  codeLanguage="typescript"
  keyPoints={["Simple REST API integration"]}
/>
```

### Image Layout

For visual-heavy slides.

```tsx
<SlideSection
  number={1}
  title="System Architecture"
  layout="image"
  imageSrc="/img/architecture.png"
  imageAlt="System architecture diagram"
  imageCaption="Edge-first processing pipeline"
  keyPoints={[]}
/>
```

## Speaker Notes

Add presenter guidance that appears in the script view and presentation mode:

```tsx
<SlideSection
  number={1}
  title="The Problem"
  keyPoints={["Key stat 1", "Key stat 2"]}
  script="Full script for the presenter..."
  speakerNotes="Pause here. Make eye contact. This is the emotional hook."
/>
```

- `script`: Full narrative for the slide
- `speakerNotes`: Quick presenter tips (separate from the full script)

## Presentation Mode

Full-screen presentation with keyboard controls:

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `‚Üí` / `Space` / `PageDown` | Next slide |
| `‚Üê` / `PageUp` | Previous slide |
| `Home` | First slide |
| `End` | Last slide |
| `1-9` | Jump to slide 1-9 |
| `N` | Toggle speaker notes |
| `T` | Toggle slide thumbnails |
| `A` | Toggle bullet animation |
| `S` | Toggle timer |
| `R` | Reset timer |
| `Escape` | Exit presentation |

### Timer & Pacing

The presentation mode includes:
- **Elapsed time** counter
- **Target time** based on slide durations
- **Pacing indicator**: Green (ahead), Yellow (on track), Red (behind)

## Charts in Slides

Use `SlideChart` for data visualization:

```tsx
import { SlideChart, createChartData } from "@site/src/components/Downloads";

// In your slide content
<SlideChart
  type="bar"
  data={[
    { label: "Q1", value: 100 },
    { label: "Q2", value: 150 },
    { label: "Q3", value: 200 },
    { label: "Q4", value: 250 }
  ]}
  title="Quarterly Growth"
  showValues={true}
  showLegend={true}
/>
```

### Chart Types

| Type | Best For |
|------|----------|
| `bar` | Comparisons, categories |
| `line` | Trends over time |
| `pie` | Part-to-whole relationships |
| `donut` | Same as pie, with center space |

### Helper Function

```tsx
// Convert object to chart data
const data = createChartData({
  "Military": 1200,
  "Infrastructure": 600,
  "Commercial": 400
});
```

## Using Data Files

Import centralized data values for consistency:

```tsx
  number={1}

  ]}
/>
```

This ensures values stay in sync across all documents when data changes.

## Print Optimization

Slides include print-optimized CSS:

- Automatic page breaks between slides
- Hidden navigation elements
- Preserved formatting and layouts
- Optimized for A4/Letter paper

### Print Tips

1. Use `Ctrl/Cmd + P` to open print dialog
2. Select "Save as PDF"
3. Enable "Background graphics" for full styling
4. Set margins to "Minimum" for best results

## Best Practices

### Content Guidelines

- **3-5 key points** per slide maximum
- **One idea** per slide
- Use **bold** for metrics and key terms
- Keep bullet text **concise** (1-2 lines)

### Visual Hierarchy

- Use icons (`üéØ`, `üí∞`, `üöÄ`) for quick recognition
- Apply two-column layout for comparisons
- Reserve title-only for section transitions

### Timing

- **30-60 seconds** per content slide
- **10-15 seconds** for section dividers
- Total duration should match intended presentation time

### Accessibility

- Always include `script` for screen reader support
- Use descriptive `title` text
- Ensure sufficient color contrast in custom themes
